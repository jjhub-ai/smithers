# 0.14.0

## New Components

- **`<Timer>`** ‚Äî Durable timer with persistent scheduling. Permits workflows to sleep for a specific relative duration or until an absolute time without keeping the internal process alive. The workflow engine will automatically resume execution when the timer fires.
  ```tsx
  <Timer id="cooldown" duration="1h" />
  <Timer id="market-open" until="2026-04-03T09:30:00Z" />
  ```
- **`<Sandbox>`** ‚Äî Execute untrusted code or compute-heavy tasks in isolated environments via Airlock bundles. Supports three distinct backend runtimes:
  - **Bubblewrap** (default): Local lightweight Linux namespace isolation.
  - **Docker**: Local container execution with configurable CPU/memory limits and volume mounts.
  - **Codeplane**: Remote VM workspace execution via SSH.
  Also includes built-in approval gates for reviewing any filesystem diffs/patches produced by the sandbox before they are applied to the host.
  ```tsx
  <Sandbox id="build" runtime="docker" image="node:22-slim" memoryLimit="2g">
    <Task id="compile" output={outputs.compiled}>Run the build</Task>
  </Sandbox>
  ```
- **`<ContinueAsNew>`** ‚Äî Replace a long-running workflow with a fresh run while carrying over required state. This explicitly prevents unbounded event-history database growth in loops and daemons. Also available as an imperative `continueAsNew()` API. Old run history remains queryable and the continuation lineage is preserved.
  ```tsx
  <Loop id="daemon" until={false} continueAsNewEvery={1000}>
    {/* loop body */}
  </Loop>
  ```
- **`<Signal>`** ‚Äî Safely inject structured, external data into a running workflow. Signals pause workflow execution until data is received via the CLI or HTTP API, validate the incoming payload against a defined Zod schema, and pass the type-safe data to downstream tasks.
  ```tsx
  <Signal id="user-feedback" schema={z.object({ rating: z.number() })}>
    {(data) => <Task id="process" input={data} />}
  </Signal>
  ```
  ```bash
  smithers signal abc123 user-feedback --data '{"rating": 5}'
  ```

## Improved Durability

- **Transactional state writes** ‚Äî Critical SQLite updates (such as task completions, approval decisions, and frame commits) are now grouped into atomic `BEGIN IMMEDIATE` transactions. This guarantees complete state consistency, preventing partial DB updates if a process crashes mid-write.
- **Task heartbeats with checkpoints** ‚Äî Granular liveness tracking per-task. Long-running tasks and remote sandbox workers can now report progress checkpoints. If the task times out or crashes, the retry execution automatically receives the final checkpoint via `lastHeartbeat` to resume work where it left off.
  ```ts
  runtime.heartbeat({ progress: 75, cursor: "page-42" });
  ```
- **Auto-resume supervisor** ‚Äî A new background loop that actively watches for orphaned runs with stale heartbeats and automatically resumes them across process crashes. Includes an atomic DB claim lock to prevent concurrent supervisor data races.
  ```bash
  smithers supervise
  ```

## Init

- **Tier-based agent groupings** ‚Äî `smithers init` now generates an `agents.ts` file with tier-based agent groupings (`cheapFast`, `smart`, `smartTool`) instead of role-based chains. Each tier is populated from detected providers with automatic fallback, and Claude Sonnet is derived as a variant for the `cheapFast` tier when Claude is available.
- **Explicit input schemas** ‚Äî All seeded workflows now define typed `inputSchema` objects passed to `createSmithers()`, enabling input validation and better IDE autocompletion.
- **ValidationLoop with feedback loop** ‚Äî The `ValidationLoop` component now wraps its sequence in a `<Loop>` with configurable `feedback`, `done`, and `maxIterations` props. Previous attempt feedback is injected into the implement prompt, and tasks include `timeoutMs` / `heartbeatTimeoutMs` for long-running agents.
- **GrillMe rewrite** ‚Äî The `GrillMe` component now imports the `grill-me` skill definition and the new `ask-user-instructions.mdx` prompt. It accepts `output`, `currentDraft`, `children`, and `until` props for composable convergence loops.
- **write-a-prd convergence** ‚Äî The `write-a-prd` workflow now composes `GrillMe` with a nested PRD task, iterating until the draft stabilizes (consecutive identical outputs). The PRD schema adds `observabilityRequirements`, `metrics`, and `verificationStrategy` fields.
- **ticket-implement feedback** ‚Äî The `ticket-implement` workflow now reads prior validation and review outputs to build a feedback string, passing `feedback`, `done`, and `maxIterations` to `ValidationLoop` for self-correcting iterations.
- **New workflow: ticket-kanban** ‚Äî Implements all tickets from `.smithers/tickets/` in parallel worktree branches, then merges completed branches back into main. Each ticket runs through the full `ValidationLoop` with feedback, and a final merge task resolves conflicts.
- **New prompt: ask-user-instructions.mdx** ‚Äî Instructions for agents to ask the user interactive questions via `bun .smithers/scripts/ask-user.ts`.
- **zod/v4** ‚Äî All seeded files now import from `zod/v4`.
- **`smithers workflow create`** ‚Äî The scaffold template for new workflows also uses the tier-based agent system and explicit input schemas.

## CLI

- **`smithers why <run-id>`** ‚Äî A powerful diagnostic engine that explains exactly why a run is stuck. Calculates blockers (e.g., waiting signals, pending approvals, stale heartbeats, retry backoffs, or upstream dependency failures) and delivers an actionable CTA to unblock it.
  ```bash
  $ smithers why abc123
  Run abc123 is waiting-approval
    Blocked node:  review-gate
    Reason:        Approval requested ‚Äî no decision yet
    Unblock:       smithers approve abc123
  ```
- **`smithers events <run-id>`** ‚Äî Browse and filter a structured workflow event history directly from the CLI. Filter by node (`--node <id>`), category (`--type <category>`), or time (`--since <duration>`), and safely stream events incrementally without running out of memory. Great for debugging the timeline of a run.
  ```bash
  smithers events abc123 --type approval --since 5m
  ```
- **`smithers node <id> -r <runId>`** ‚Äî Provides deep, enriched inspectability into an individual node. It cleanly aggregates task attempts, executed tool calls, schema validation errors, Zod-validated outputs, and tracks full token usage and pricing.
  ```bash
  smithers node review-step -r abc123 --tools --attempts
  ```
- **`--watch` flag** ‚Äî Live-updating read views for the `ps`, `inspect`, and `events` commands.
  ```bash
  smithers ps --watch --interval 2
  ```

## Engine & Storage

- **Frame delta encoding** ‚Äî A massive 95‚Äì99% reduction in `_smithers_frames` SQLite storage space. Time-travel rendering frames are now stored as compact XML structural diffs against the previous frame instead of full snapshots, effectively shrinking multi-gigabyte workflow databases down to megabytes.

## New Run Statuses

- `waiting-timer` ‚Äî run is paused on a durable timer
- `continued` ‚Äî run completed via continue-as-new and a successor run was started
