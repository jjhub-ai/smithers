Smithers has separate installation guides for each authoring API.

## Pick an Installation Guide

| API | Use this when | Installation guide |
| --- | --- | --- |
| **TOON** | You want declarative `.toon` files and the least setup | [TOON Installation](/toon/installation) |
| **JSX** | You want TSX workflows, MDX prompts, and render-time branching | [JSX Installation](/jsx/installation) |

## Shared Requirements

Both APIs run on the same underlying runtime, so these show up everywhere:

- Bun
- local SQLite persistence
- optional model or provider credentials
- the shared runtime docs for observability, CLI usage, and integrations
