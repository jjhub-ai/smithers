Smithers has a different quickstart for each authoring API.

## Pick a Quickstart

| API | What you build | Quickstart |
| --- | --- | --- |
| **TOON** | A declarative `.toon` workflow with inline schemas and prompts | [TOON Quickstart](/toon/quickstart) |
| **JSX** | A TSX workflow with `<Task>` components and `ctx.outputMaybe(...)` | [JSX Quickstart](/jsx/quickstart) |
| **Effect Builder** | A low-level workflow graph built with `$.step()` and `Layer` | [Effect Quickstart](/effect/quickstart) |

## Same Runtime, Different Authoring Style

All three quickstarts end up on the same runtime:

- steps are persisted to SQLite
- completed work resumes after restarts
- retries, approvals, and loops use the same engine
- observability and CLI tooling are shared
