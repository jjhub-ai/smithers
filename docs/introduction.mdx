Smithers supports two authoring APIs. They both compile to the same durable execution engine, but they optimize for different ways of building workflows.

## Choose an API

| API | Best for | Start here |
| --- | --- | --- |
| **TOON** | Prompt-first workflows and the fastest path to a working flow | [Overview](/toon/overview), [Installation](/toon/installation), [Quickstart](/toon/quickstart) |
| **JSX** | TSX composition, render-time branching, and MDX prompts | [Overview](/jsx/overview), [Installation](/jsx/installation), [Quickstart](/jsx/quickstart) |

## Shared Runtime

No matter which API you choose, Smithers handles the same runtime concerns:

- durable execution and resumability
- retries, timeouts, and approval gates
- SQLite persistence
- parallel execution and loops
- structured events, metrics, and observability

The sidebar sections for Core Concepts, Runtime, CLI, and Integrations are shared across both API tabs because they describe the common engine underneath the authoring layers.
