Prompts are the core of most TOON workflows. The `prompt:` field on a step defines the text sent to the step's agent.

<Note>
  Every `prompt:` step requires an `agent:` field referencing a declared agent. The examples below omit `agents:` declarations and `agent:` fields for brevity — see [Nodes](/toon/nodes) for complete examples.
</Note>

## Basic Prompt

```toon
steps:
  - id: greet
    prompt: Say hello to {input.name}.
    output:
      message: string
```

## Multi-line Prompts

Use YAML's `|` (literal block) syntax for multi-line prompts:

```toon
steps:
  - id: analyze
    prompt: |
      You are a senior software engineer.

      Analyze the following bug report and identify:
      1. Root cause
      2. Affected systems
      3. Suggested fix

      Bug report:
      {input.description}
    output:
      rootCause: string
      affectedSystems: string[]
      suggestedFix: string
```

The `|` preserves newlines. Each line is included exactly as written.

## Interpolation

Use `{expression}` to inject values into prompts:

### Workflow Input

```toon
prompt: |
  Research the topic: {input.topic}
  Priority: {input.priority}
```

### Upstream Step Outputs

```toon
prompt: |
  Based on the analysis:
  Summary: {analyze.summary}
  Severity: {analyze.severity}
```

### Nested Fields

```toon
prompt: |
  Deploy to {config.deployment.environment}
  Region: {config.deployment.region}
```

### Array Values

Arrays are interpolated as comma-separated values by default:

```toon
prompt: |
  The key points are: {research.keyPoints}
```

For more control, reference individual items or use them in a list context:

```toon
prompt: |
  Key findings:
  {research.keyPoints}

  Please elaborate on each point above.
```

## Escaping Braces

Use double braces to output a literal brace:

```toon
prompt: |
  Return a JSON object like {{key: value}}.
  The input topic is {input.topic}.
```

`{{` becomes `{` and `}}` becomes `}` in the final prompt text.

## Conditional Sections

Use `{#if}` blocks for conditional prompt sections:

```toon
steps:
  - id: fix
    prompt: |
      Fix the bug described below.

      {#if input.priority == "high"}
      URGENT: This is a high-priority fix. Minimize the blast radius.
      {/if}

      Description: {input.description}
    output:
      patch: string
```

## Referencing Loop State

Inside a loop, reference the current iteration's data:

```toon
steps:
  - kind: loop
    until: "{review.approved}"
    children:
      - id: review
        prompt: |
          Review this content.
          This is iteration {loop.iteration}.
          {#if loop.iteration > 1}
          Previous feedback: {review.feedback}
          {/if}
        output:
          approved: boolean
          feedback: string
```

## Component Parameters in Prompts

Inside a component, use `{params.fieldName}`:

```toon
components:
  Reviewer:
    params:
      content: string
      role: string
    steps:
      - id: "{id}-review"
        prompt: |
          You are {params.role}.
          Review: {params.content}
        output:
          approved: boolean
          feedback: string
```

## Prompt Best Practices

**Keep prompts focused.** Each step should have a single clear instruction. Split complex tasks across multiple steps.

**Use interpolation for data, not logic.** If you need complex string manipulation, use an inline `run:` block to prepare the data, then reference its output in a downstream prompt.

**Leverage multi-line.** The `|` syntax makes prompts readable. Avoid cramming everything onto one line.

**Reference specific fields.** Instead of passing an entire object, reference the specific fields the model needs:

```toon
# Prefer this
prompt: |
  Summary: {analysis.summary}
  Severity: {analysis.severity}

# Over this
prompt: |
  Analysis: {analysis}
```
